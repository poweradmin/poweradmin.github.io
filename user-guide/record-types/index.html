DNS Record Types

Overview

DNS record types determine how your domain name is resolved and what services are available. Poweradmin supports a wide range of record types for both forward and reverse DNS zones.

Note: Administrators can customize which record types are available in your Poweradmin installation. Contact your system administrator if you need a specific record type that isn't available in your interface. For details on customizing available record types, see Record Type Customization.

Commonly Used Record Types

Record Type Description Example
A Maps a domain name to an IPv4 address example.com. IN A 192.0.2.1
AAAA Maps a domain name to an IPv6 address example.com. IN AAAA 2001:db8::1
CNAME Creates an alias for another domain name www.example.com. IN CNAME example.com.
MX Specifies mail servers for the domain example.com. IN MX 10 mail.example.com.
NS Specifies name servers for the domain example.com. IN NS ns1.example.com.
PTR Maps an IP address to a domain name (reverse lookup) 1.2.0.192.in-addr.arpa. IN PTR example.com.
SOA Start of Authority, contains administrative information example.com. IN SOA ns1.example.com. hostmaster.example.com. (2023010101 3600 1800 604800 86400)
TXT Contains text information (often used for verification) example.com. IN TXT "v=spf1 include:_spf.example.com ~all"

Security and Authentication Records

Record Type Description When to Use
CAA Certificate Authority Authorization Specify which CA can issue SSL certificates for your domain
DNSKEY Public key for DNSSEC Part of DNSSEC implementation
DS Delegation Signer Hash of a DNSKEY record, used in DNSSEC
SSHFP SSH Public Key Fingerprint Verify SSH host keys via DNS
TLSA TLS Authentication Specify certificate information for TLS verification

Service Records

Record Type Description When to Use
SRV Service Locator Define hostname and port for specific services
NAPTR Name Authority Pointer Used for ENUM and other transformations

Adding Records

When adding DNS records in Poweradmin:

  1. Navigate to the zone where you want to add the record
  2. Click on "Add Record"
  3. Select the record type from the dropdown menu
  4. Fill in the required fields for that record type
  5. Click "Add Record" to save

Special Considerations

SOA Records

  • SOA records are automatically created when you create a new zone
  • Only one SOA record can exist per zone
  • Modifying SOA records should be done with care as they control zone transfers and refresh rates

NS Records

  • At least two NS records are recommended for redundancy
  • NS records should point to actual, operational name servers

MX Records

  • Include a priority value (lower numbers have higher priority)
  • Mail servers referenced should have valid A or AAAA records

CNAME Records

  • Cannot coexist with other records of the same name (except DNSSEC records)
  • Should not be used for the zone apex (root domain)

TXT Records

  • May need special formatting or quoting depending on content
  • Commonly used for domain verification, SPF, DKIM, etc.

Reverse DNS Records

Reverse DNS mappings (PTR records) are created in special reverse zones: - For IPv4: [octet].[octet].[octet].[octet].in-addr.arpa - For IPv6: [nibble].[nibble]....[nibble].ip6.arpa

Zone Template Records

When creating zone templates, you can define standard records that will be automatically created whenever a zone is created from that template.
